# ⚡ 智能电费监控系统 (Smart Electricity Monitor)

## 📖 应用场景

智慧工大APP缺乏实时的“电费余额不足”提醒功能。这常常导致我们在毫无防备的情况下遭遇突然欠费宿舍停电的窘境，影响正常的生活和学习。

为了解决这一痛点，本项目提供了一个**自动化电费余额监测服务**。系统会根据配置的策略，定时或根据电费余额动态智能调整查询频率（余额充足时降低频率，避免高频请求被封；余额见底时进入高频监控），并在电费跌破设定的警戒线时，主动通过邮件发送精美的 HTML 告警提醒您及时充值，彻底告别突然断电的烦恼！

## ⚙️ 配置教程

系统运行依赖于配置文件 `config.yaml`。请参照以下步骤完成设定：

1. **复制配置文件模板**  
   在项目根目录下，将 `example.config.yaml` 复制一份并重命名为 `config.yaml`。
   
2. **详细配置参数说明**  
   打开 `config.yaml` 文件，根据以下说明详细填写每一项配置：

   ### 账号配置 (`account`)
   - `username`: **必填**。您的智慧工大账号（通常为学号）。
   - `password`: **必填**。您的智慧工大登录密码。

   ### 邮件推送消息配置 (`email`)
   - `smtp_server`: **必填**。SMTP服务器地址（默认可选其他服务商，推荐 QQ 邮箱的 `smtp.qq.com`）。
   - `smtp_port`: **必填**。SMTP端口号（通常 QQ 邮箱使用 `465` 端口）。
   - `sender_email`: **必填**。发件人邮箱地址（填使用您自己的 QQ 邮箱）。
   - `sender_password`: **必填**。发件人邮箱的 SMTP 密钥/授权码（注意：不是您的邮箱登录密码。SMTP 密钥需要单独申请，具体获取步骤请自行上网搜索“QQ邮箱开启SMTP服务并获取授权码”）。
   - `receiver_email`: **必填**。收件人邮箱地址（接收告警的邮箱，可以使用与发件人相同的 QQ 邮箱地址，目前系统只能配置接收一个）。

   ### 监控与阈值配置 (`monitor`)
   - `alert_threshold`: **必填**。电费告警阈值（单位：元）。当账户余额低于此设定值时，系统将会触发邮件提醒。例如：`5`。
   - `max_interval_hours`: **必填**。最大查询间隔时间（单位：小时）。即系统的最低查询频率，不管电量多充足，至少每隔这么久查询一次。例如：`12.0`（半天测一次）。
   - `min_interval_hours`: **必填**。最小间隔时间（单位：小时）。即余额快见底时，系统的最高查询频率限制，防止过于频繁请求被封禁账号或IP。例如：`0.5`（半小时查一次）。
   - `safe_margin`: **必填**。安全边际系数（取值范围：`0-1`，默认推荐 `0.8`）。控制算法预测查询的提前量。如果算法预测还能用 10 小时，当设为 0.8 时，会在第 8 小时就提前进行余额复查，以确保绝对安全。
   - `email_cooldown`: **必填**。邮箱冷却时间（单位：秒）。即连续触发告警的情况下，两次发送邮件之间的最小间隔。例如设为 `7200` 代表 2 小时内部重复发送告警邮件。

## 🚀 启动教程

> 💡 **强烈推荐**: 请将本服务部署在一台**永不关机的服务器**（如云服务器、树莓派或一直运行的软路由等设备）上，以确保监控服务能够 7x24 小时不间断运行。

### 环境准备

本项目要求 Python 版本 >= 3.11。
项目使用 [uv](https://docs.astral.sh/uv/) 作为极速的 Python 包管理器和环境分配工具。NodeJS (用于安装 pm2) 若要在后台持久化运行也需要准备。

### 安装依赖并运行

**方法 1：使用 uv 快速启动（推荐）**

如果您已经安装了 `uv`，只需在项目目录下执行：
```bash
uv run main.py
```
这会自动创建虚拟环境、无需处理系统范围冲突、安装依赖并直接启动服务。  
如果您还没有安装 `uv`，可以参考官方教程安装（例如 Linux 环境下）：
```bash
curl -LsSf https://astral.sh/uv/install.sh | sh
```

**方法 2：使用传统 pip 启动**

如果您没有安装 `uv` 也不想处理环境隔离，也可以运行标准的 pip 安装依赖并直接从 Python 运行：
```bash
pip install -r pyproject.toml # 或者手动安装下方库
pip install apscheduler pyyaml requests
python main.py
```

### 持续在服务器后台运行 (Linux 案例) 

为了在断开 SSH 连接后服务继续稳定运行、自动重启和管理日志，**强烈推荐使用 `pm2`** 作为进程管理工具。

**步骤 1: 全局安装 pm2**

如果您的服务器还没有安装 npm 和 pm2：
```bash
# Ubuntu/Debian系统 示例
sudo apt update
sudo apt install nodejs npm
# 全局安装 pm2
sudo npm install -g pm2
```

**步骤 2: 使用 pm2 启动监控服务**

无论您使用的是 `uv` 还是普通的 `python`，都可以用 `pm2` 托管进程。

如果您使用 `uv` 运行项目（假设您在项目根目录）：
```bash
# 运行进程并将其命名为 ele-monitor
pm2 start "uv run main.py" --name "ele-monitor"
```

如果您使用传统 `python` 命令：
```bash
pm2 start main.py --name "ele-monitor" --interpreter python3
```

**步骤 3: 常用 pm2 管理命令**
- **查看状态**: `pm2 status`
- **查看实时日志**: `pm2 logs ele-monitor`
- **重启服务**: `pm2 restart ele-monitor`
- **停止服务**: `pm2 stop ele-monitor`
- **删除后台任务**: `pm2 delete ele-monitor`

**添加开机自启（可选但推荐）**:
为了服务器重启后依然能自动恢复程序，可执行：
```bash
pm2 startup
pm2 save
```
